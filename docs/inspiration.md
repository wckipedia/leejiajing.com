# Inspiration

This log records references and, more importantly, the reason each reference
matters. Do not copy a reference wholesale.

## Current-site observations

- The existing grid and warm neutral palette provide consistency, but the new
  direction should not be constrained by them.
- The hero establishes a personal identity quickly, while the project evidence
  begins too far below the first viewport.
- Project data contains useful technical depth, but supporting pages still include
  placeholder or generic copy.
- The current cards and timeline are reusable ideas; their visual treatment and
  information hierarchy can be reconsidered completely.
- Contact and resume actions already have clear utility and should remain easy to find.

## Portfolio-specific design observations

- A recruiter should understand role, strengths, and strongest work within one minute.
- An engineering manager should find ownership, constraints, tradeoffs, and outcomes.
- A technical interviewer should be able to follow implementation decisions into
  deeper project detail.
- “Premium” should come from precision, writing, typography, and interaction quality,
  not from visual effects alone.
- The memorable idea should reinforce engineering identity rather than resemble a
  SaaS launch page.
