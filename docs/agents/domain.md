# Domain Documentation

This is a single-context repository.

Engineering skills should read these sources when they exist:

- `CONTEXT.md` at the repository root for shared domain language.
- `docs/adr/` for accepted architectural decisions.

Missing domain files are not an error. Create them only when the project develops
durable terminology or an architectural decision worth preserving.

Use terms from `CONTEXT.md` consistently in issues, code, tests, and review notes.
If a proposed change conflicts with an ADR, identify the conflict explicitly
instead of silently overriding the decision.
